## Why

Writing complex crawling rules in raw JSON with nested arrays (like `["@css:.video", "@text", "@trim"]`) is unintuitive and error-prone for users. Spectra needs a visually driven, node-based blueprint editor (similar to n8n or Unreal Engine Blueprints) to allow users to drag, drop, and wire together rule pipelines intuitively, along with a live WebView preview to test extractions instantly.

## What Changes

- **React Flow Editor Layer**: Implements a web-based UI utilizing `@xyflow/react` to render `PipelineNode` definitions as visual blocks.
- **WebView Element Picker**: Adds a live preview panel allowing users to click Dom elements on a target page to automatically generate CSS/XPath selectors.
- **Live Serialization**: Automatically syncs the visual node graph into Spectra's unified JSON format for the Rust parser.

## Capabilities

### New Capabilities
- `visual-rule-editor`: The entire web-based visual workspace for creating, editing, and previewing unified crawling rules without writing raw JSON.
- `element-picker`: The visual selector logic that translates DOM clicks into valid `@css` or `@xpath` pipeline nodes.

### Modified Capabilities
- `web-editor`: The existing simple form editor is completely replaced by this node-flow architecture.

## Impact

- Rewrites the `web-editor/` frontend utilizing React, Zustand, and Tailwind.
- Ingests the strongly-typed interfaces generated by the `cross-language-rule-model` change (`ts-rs`).
